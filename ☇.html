my $repo = Git->repository (Directory => $dir);
my @filespec = split(/ /,$filespec);
my @revs = $repo->command('rev-list', '--all', '--', @filespec);
my @data = ("Lines changed");
my $prev_commit = pop @revs; #Throwaway first commit
$prev_commit = pop @revs;
for my $commit ( reverse @revs ) {
  my $file_contents = $repo->command('diff','--shortstat',  "$commit..$prev_commit", "--", @filespec );
  my ($insertions) = ($file_contents =~ /(\d+)\s+insertion\S+/s);
  my ($deletions) = ($file_contents =~ /(\d+)\s+deletion\S+/s);
  $insertions = $insertions || 0; $deletions = $deletions || 0;
  my $lines_changed = ($insertions > $deletions)?$insertions:$deletions;
  push @data, $lines_changed;
  $prev_commit = $commit;
}

Explaining how this might work. There are no camel ants. There are camel spiders that feed on ants. And the other way round, if there are enough of them. That software development self-organizes was explained in a model by us in '05 in this paper. This presentation does more or less the same for non-software writing and using somewhat different measures and techniques. Goes to prove the same, anyway. Stigmergy is now understood as an essential model for modern, distributed, software development. Stigmergy is a general mechanism for collaboration. And there's no such a thing as camel ants, but there are fire ants, and "fire" is "llama" in Spanish. So there you have it.
